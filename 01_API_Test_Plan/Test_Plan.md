# API Test Plan

## 1. Introduction
This document describes the API testing strategy for a demo REST API application.

## 2. Objective
To validate that API endpoints function correctly, return proper status codes, and handle valid and invalid data appropriately.

## 3. Scope
The following endpoints will be tested:
- GET /users
- GET /users/{id}
- POST /users
- PUT /users/{id}
- DELETE /users/{id}

## 4. Testing Types
- Functional Testing
- Negative Testing
- Status Code Validation
- Response Body Validation

## 5. Tools Used
- Postman
- GitHub

## 6. Test Environment
Base URL:
https://reqres.in/api

## 7. Deliverables
- Test Plan
- API Scenarios
- Postman Collection
- Test Execution Report
